ThreatMapper v2.0: Self-Hosted AI Threat Intelligence Platform

ThreatMapper turns threat reports, DFIR writeups, and investigation notes into evidence-reviewed MITRE ATT&CK mappings, then connects those mappings to actor/campaign comparison, Navigator layers, PDF reporting, and STIX/OpenCTI export.

The Problem

CTI reports are useful, but they are often difficult to operationalize. Analysts still need to extract behavior, map it to ATT&CK, validate evidence, compare it with known actor and campaign profiles, and turn the result into hunting or detection work.

ThreatMapper v2.0 is built for that workflow. It is not an attribution engine and it is not an IOC feed. It is a self-hosted analyst workbench for converting report text into reviewed, defensible TTP mappings.

What ThreatMapper Is

ThreatMapper is an open-source CTI-to-detection workbench. The full Docker platform supports private report analysis with Claude, OpenAI, Gemini, or a local OpenAI-compatible LLM. The public browser workspace supports ATT&CK exploration, manual layers, group overlays, and browser-side comparison.

Self-Hosted Architecture

The Docker deployment uses a React/Vite frontend, FastAPI backend, PostgreSQL, Redis, Celery workers, and a scheduled ATT&CK synchronization pipeline. Report content stays inside the operator's deployment except for the selected LLM provider call.

AI Analysis With Review States

The AI Analysis page accepts pasted text, PDF, DOCX, and TXT files. Each extracted mapping includes ATT&CK technique ID, technique name, tactic, confidence, evidence, evidence source, and review state.

Review states separate model output from analyst-approved findings: suggested, accepted, rejected, and needs-evidence.

Navigator Handoff

Reviewed techniques can be injected into the Navigator workspace. The matrix supports Enterprise, Mobile, and ICS ATT&CK domains, selected TTP layers, overlays, search, platform filtering, imports, and exports.

Actor, Campaign, and Report Comparison

ThreatMapper compares selected TTPs against ATT&CK group profiles, campaign profiles, and previous locally stored analyses. Similarity is useful for prioritization and investigation leads, but it is not attribution proof.

DFIR Examples

ThreatMapper includes indexed public DFIR Report metadata for practical testing. It does not mirror third-party report text or artifacts; users open the original report and save a local PDF for private analysis.

PDF and STIX/OpenCTI Export

Completed analyses can be exported as PDF reports for analyst handoff and as STIX 2.1 bundles for OpenCTI workflows. The STIX export models TTP evidence and similarity leads; it is intentionally not an IOC-centric export.

Recommended Workflow

  1. Start with an authorized report or DFIR writeup.
  2. Run AI Analysis with your configured provider or local LLM.
  3. Review every extracted technique and reject unsupported mappings.
  4. Inject accepted TTPs into Navigator.
  5. Compare against groups, campaigns, and previous reports.
  6. Identify gaps and detection engineering priorities.
  7. Export PDF or STIX/OpenCTI output for handoff.

Important Limitation

ThreatMapper assists analysts; it does not replace analyst validation. ATT&CK overlap, actor names, shared tools, and model output are investigation signals. They are not final attribution evidence.
