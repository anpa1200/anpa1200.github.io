Full-version boundary guide

Feature guides for gated AdversaryGraph modules.

Threat Matrix Light shows the same product structure as AdversaryGraph, but browser-only pages cannot run private analysis, feed sync, AI providers, RAG retrieval, malware workflows, attack simulation, or SIEM forwarding. Each section below explains the full-version workflow and links the light button to a documented operator path.

Screenshots are reused from the published AdversaryGraph documentation and release evidence set. Validate commands, credentials, and SIEM field names in your own self-hosted deployment.

AI Analyze

Use this module to turn reports, logs, PDFs, and analyst notes into reviewed ATT&CK mappings, IOC leads, hypotheses, and report-ready investigation context.

  1. Upload or paste the source material in the self-hosted workspace.
  2. Select the permitted AI provider and classification boundary.
  3. Review extracted entities, TTPs, assumptions, and evidence gaps.
  4. Accept only reviewed fields into the investigation.

The public page never sends uploaded content to an AI provider.

Reports Research

Store report sessions and compare them against actors, techniques, campaigns, and previous investigations.

  1. Create a report research session.
  2. Attach the source report and extracted entities.
  3. Compare behavior against ATT&CK groups and campaign context.
  4. Export a reviewed report or handoff package.

Evidence Graph

Persist reviewed links between IOCs, files, reports, actors, CVEs, techniques, hunts, and operational cases.

  1. Open the investigation workspace.
  2. Add reviewed nodes from analysis or enrichment results.
  3. Connect evidence edges with source notes.
  4. Use the graph for briefing, export, and follow-up tasks.

Operations

Use operations to manage cases, tasks, status, ownership, and handoff across CTI, detection, hunting, and validation work.

  1. Create a case or attach to an existing operation.
  2. Assign tasks and evidence requirements.
  3. Track analyst decisions and lifecycle state.
  4. Export the operation summary for review.

Pipeline

The pipeline module automates controlled ingestion, sync, normalization, analysis, export, and validation jobs.

  1. Select source feeds and job type.
  2. Run or schedule the pipeline.
  3. Review stage logs and failures.
  4. Promote only validated output to production workflows.
Pipeline guide

Observability

Observability covers self-test, health checks, source freshness, storage permissions, API configuration, and release-readiness evidence.

  1. Run platform self-test after deployment or update.
  2. Review degraded sources separately from core service health.
  3. Inspect logs and remediation guidance.
  4. Recheck after configuration changes.
Observability guide

Threat Radar

Threat Radar scores actor, campaign, IOC, CVE, and TTP relevance against business context such as geography, sector, exposed technology, and priority assets.

  1. Define business context and monitored assets.
  2. Run source-backed relevance scoring.
  3. Review matched threats and supporting evidence.
  4. Promote approved leads to hunts or operations.
RAG and relevance guide

Asset Surface

Asset Surface connects asset inventories with CVE, IOC, and TTP intelligence so analysts can prioritize what matters to the environment.

  1. Import asset inventory and business labels.
  2. Normalize software, host, network, and ownership fields.
  3. Match CVEs, IOCs, and actor behavior to assets.
  4. Export prioritized remediation and hunt leads.
CVE intelligence guide

Threat Hunting

Threat Hunting manages hypotheses, ATT&CK scope, query drafts, evidence review, and outcome handoff without claiming that a public web page executed telemetry searches.

  1. Create a hypothesis from a report, IOC, CVE, actor, or technique.
  2. Choose target telemetry and query language.
  3. Use AI assistant output as a draft only.
  4. Validate syntax in the approved SIEM/EDR before recording findings.

Query Library

The full query library stores Sigma, SPL, KQL, EQL, YARA, and YARA-L examples with tags, TTP mappings, provenance, and review state.

  1. Search by ATT&CK ID, telemetry source, platform, or tag.
  2. Generate or adapt a query from a reviewed hypothesis.
  3. Record source links and assumptions.
  4. Copy only reviewed output into approved telemetry tools.
Detection content guide

IOC Library

The full IOC Library stores feed-backed indicators with deduplication, normalization, source history, tags, and enrichment state.

  1. Connect public or private IOC feeds.
  2. Run sync and review source status.
  3. Search indicators by value, tag, source, family, or first-seen time.
  4. Promote relevant IOCs to investigations or exports.
IOC Library guideAdd custom feed

IOC Investigation

IOC Investigation enriches an indicator through configured providers such as VirusTotal, OTX, urlscan, Shodan, GreyNoise, OpenCTI, and local data.

  1. Paste or select an IOC.
  2. Run only configured and permitted enrichment providers.
  3. Review source results and confidence.
  4. Add confirmed findings to the evidence graph.

CVE Intelligence

CVE Intelligence prioritizes vulnerabilities with KEV, CVSS, EPSS, exploit context, asset exposure, and remediation state.

  1. Sync CVE sources and verify freshness.
  2. Filter by vendor, product, exploit status, ransomware use, and asset exposure.
  3. Review affected assets and exploit notes.
  4. Create remediation, hunt, or validation tasks.
CVE Library guide

Feeds Management

Feeds Management controls public/private source configuration, sync status, degradation handling, and source-specific troubleshooting.

  1. Configure feed URL, token, and enablement policy.
  2. Run sync and inspect count, last sync, and errors.
  3. Repair unreachable sources or disable intentionally unavailable feeds.
  4. Confirm deduplication and normalization in self-test.

Attack Simulation

Attack Simulation creates controlled validation scenarios for detection rules by mapping TTPs to lab actions, expected telemetry, and SIEM forwarding.

  1. Select ATT&CK techniques or generate an attack chain.
  2. Configure lab target, safety controls, and event forwarding.
  3. Run the simulation in the controlled lab.
  4. Compare expected and observed rule hits.
Attack simulation guide

RetroHunt

RetroHunt runs reviewed hypotheses against historical telemetry or stored datasets to verify whether behavior existed before the active investigation window.

  1. Select the hypothesis, timeframe, and data source.
  2. Attach a reviewed query or rule.
  3. Run the approved historical job.
  4. Record findings, false positives, and next actions.
Evaluation guide

Malware Analysis

MalwareGraph supports static and dynamic analysis workflows, strings, function graphs, behavior extraction, and IOC/TTP reporting.

  1. Upload a sample only in an approved isolated environment.
  2. Run static analysis and extract strings, imports, IOCs, and behaviors.
  3. Review AI suggestions as leads only.
  4. Promote confirmed behavior to reports, evidence, and detections.
Malware analysis guide

Malware Unpacker

The unpacker workflow helps analysts identify packing, extract strings, review sections, and document unpacking decisions.

  1. Open the sample in MalwareGraph.
  2. Inspect sections, entropy, strings, and packing indicators.
  3. Run unpacking actions in the controlled environment.
  4. Record recovered IOCs and behavior.
Malware analysis guide

Dynamic Analysis

Dynamic Analysis executes samples only in a controlled sandbox/lab and records behavior, network activity, runtime events, and extracted evidence.

  1. Confirm sample handling policy and lab isolation.
  2. Start a runtime session with explicit limits.
  3. Capture behavior, network, and process evidence.
  4. Stop the run and review results before export.
MalwareGraph guide

Knowledge Library

The Knowledge Library manages local source material for RAG and analyst retrieval: reports, docs, IOCs, CVEs, ATT&CK, hunts, evidence, and assets.

  1. Define allowed corpus sources and classification rules.
  2. Ingest documents and structured records.
  3. Run vector indexing locally or through the approved provider.
  4. Use RAG answers as cited leads that require human review.

Statistics

Statistics summarizes stored ATT&CK coverage, IOC/CVE volume, sync status, source freshness, investigation counts, and data quality indicators.

  1. Open the Statistics module after sync or import.
  2. Review totals, source distribution, and stale data warnings.
  3. Drill into outliers before using metrics in reports.
  4. Re-run self-test after major imports.
Validation and observability

Troubleshooting

Troubleshooting provides operator diagnostics for Docker, database, Redis, storage permissions, API keys, feed sync, ATLAS/ATT&CK data, and service health.

  1. Run self-test and open degraded checks.
  2. Read the specific remediation text for each failed source.
  3. Check logs and configuration from the self-hosted host.
  4. Re-run self-test and preserve output as release evidence.
